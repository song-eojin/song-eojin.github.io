###### ✅ 쿠키(Cookie)

쿠키는 웹 브라우징 경험을 개선하고 사용자를 식별하는 데 사용되는 작은 데이터 조각으로, 대부분 텍스트 형식으로 저장되며, 웹사이트와 웹 브라우저 간에 정보를 주고 받을 때 사용됩니다. 쿠키가 동작하는 주요 단계는 다음과 같습니다.

<div style="background-color: RGB(1,1,1, 0.02); border: 1px solid whitesmoke; padding: 20px;"><p>
1) 서버에서 쿠키 생성 : 웹 서버는 클라이언트(웹 브라우저)에게 쿠키를 생성하고 클라이언트로 전송합니다. 쿠키는 이름, 값, 만료 날짜, 도메인, 경로 및 안전 플래그와 같은 정보로 구성됩니다.<br><br>
2) 클라이언트의 요청과 함께 쿠키 전송 : 클라이언트는 웹사이트를 방문할 때 쿠키를 포함한 HTTP 요청을 웹 서버에 보냅니다. 요청 헤더에 쿠키가 포함되어 있으며, 웹 서버는 이를 확인하고 해당 세션 또는 사용자에 대한 정보를 추출합니다.<br><br>
3) 서버에서 쿠키 읽기 : 웹 서버는 클라이언트로부터 수신한 요청 헤더에 포함된 쿠키를 읽습니다. 서버는 쿠키를 사용하여 사용자의 상태를 확인하거나 개인화된 콘텐츠를 제공하는 데 필요한 데이터를 검색합니다.<br><br>
4) 서버에서 쿠키 업데이트 또는 새로운 쿠키 생성 : 웹 서버는 필요한 경우 클라이언트로부터 수신한 쿠키를 업데이트하거나 새로운 쿠키를 생성합니다. 업데이트된 쿠키는 다음 요청 때 클라이언트로 전송됩니다. <br><br>
5) 클라이언트에 쿠키 저장 : 클라이언트는 서버로부터 수신한 쿠키를 로컬에 저장합니다. 쿠키는 클라이언트 측에서 브라우저의 쿠키 저장소 또는 로컬 스토리지에 저장됩니다. 쿠키에는 설정된 만료 날짜 또는 세션 종료 시간까지 저장될 때까지 유지되는 기간이 있습니다. <br><br>
6) 다음 요청에서 쿠키 재전송 : 클라이언트가 동일한 웹사이트를 다시 방문할 때, 이전에 저장한 쿠키가 쿠키 헤더에 자동으로 포함됩니다. 이로써 서버는 사용자를 식별하고 이전 상태 정보를 유지하거나 개인화된 경험을 제공할 수 있습니다.
</p></div>

<br>

<div style="background-color: RGB(1,1,1, 0.02); border: 1px solid whitesmoke; padding: 20px;"><p>

<b>쿠키의 유형에 해당하는 지속적인 쿠키(Persistent Cookies)와 세션 쿠키(Session Cookies)의 개념과 차이점은 ❔</b><br><br>

<mark>쿠키가 얼마 동안 유지되는지</mark>에 따라 쿠키도 지속적인 쿠키와 세션 쿠키로 분류됩니다. 지속적인 쿠키는 만료 날짜를 설정하여 일정 기간 동안 유지될 수 있으며, 사용자가 로그아웃하거나 브라우저를 종료해도 계속해서 유지됩니다. 즉, <mark>지속적인 쿠키는 브라우저 세션이 끝나더라도 삭제되지 않는다</mark>는 것이 특징입니다. 따라서 주로 사용자의 환경 설정, 로그인 정보, 사용자 선호도, 추적 데이터 등을 저장하는 데 사용됩니다. 예를 들어, '로그인 유지' 옵션을 선택하면 해당 웹사이트의 로그인 정보가 지속적인 쿠키로 저장되어 사용자가 다음에 방문할 때 자동으로 로그인될 수 있습니다.<br><br>


반면 <mark>세션 쿠키는 브라우저를 닫으면 삭제</mark>됩니다. 따라서 세션 쿠키는 일반적으로 임시 데이터를 저장하는 데 사용됩니다. 사용자의 상태를 유지하거나 장바구니 내용과 같은 임시 정보를 저장하는 데 적합합니다. 사용자가 브라우저를 닫으면 세션 쿠키는 삭제되므로 다음 방문 때에는 해당 정보가 유지되지 않습니다. 예를 들어, 웹사이트에서 로그인한 후 로그인 세션을 유지하기 위해 세션 쿠키를 사용됩니다.
</p></div>

<br><br>

###### ✅ 세션(Session)

웹 애플리케이션에서 사용자와 서버 간의 상호작용을 유지하고 관리하기 위한 중요한 개념으로, 세션의 동작 방식은 다음과 같습니다.

<div style="background-color: RGB(1,1,1, 0.02); border: 1px solid whitesmoke; padding: 20px;"><p>

1) 사용자가 웹사이트에 접속하면 서버는 사용자에게 고유한 세션 식별자(Session ID)를 부여 <br>
2) 이 세션 ID는 클라이언트(웹 브라우저)에 저장되거나 쿠키에 저장<br>
3) 사용자의 모든 요청에는 해당 세션 ID가 함께 전송<br>
4) 서버는 세션 ID를 사용하여 해당 세션과 관련된 데이터를 식별하고 처리<br>
5) 사용자가 로그아웃하거나 세션 종료 조건을 충족하면 서버는 해당 세션을 종료하고 관련 데이터를 삭제

</p></div>

<hr><br>

###### Q. 쿠키와 세션의 개념과 차이점을 설명해줘 ❗<br><br>

우선 쿠키와 세션의 공통점을 먼저 살펴보겠습니다. 쿠키와 세션은 모두 웹 애플리케이션에서 사용되는 데이터 관리 기술이며, 둘다 기본적으로 HTTP 프로토콜 기반으로 동작합니다. HTTP 프로토콜을 기반으로 사용자의 상태를 관리하며, 임시 데이터를 저장하고, 웹 애플리케이션이 쿠키와 세션을 사용하여 사용자와 상호작용한다는 점에서 <mark>쿠키와 세션은 수행되어지는 역할이 같다</mark>고 볼 수 있습니다.<br><br>

<mark>하지만 쿠키와 세션이 가지는 특성의 차이에 따라 용도가 달라집니다.</mark> 이러한 차이점은 일단 저장되는 위치가 다르다는 것에서부터 시작합니다. 쿠키는 웹 브라우저 즉, 클라이언트 측에 저장되지만, 세션은 서버 측에 저장되어 세션 데이터는 클라리언트 측에서 직접 수정이 불가해 일반적으로 더 안전하다는 특징을 가집니다. 즉, 쿠키는 클라리언트 측에서 수정이 가능하기 때문에 중요한 보안 정보를 쿠키로 저장할 때에는 주의가 필요함을 의미합니다.<br><br>

또한 쿠키와 세션에는 수명에도 차이가 있습니다. 쿠키는 클라이언트 측에서 설정한 만료 날짜나 세션 종료 시점까지 계속 지속되지만, 세션은 일반적으로 사용자가 로그인한 후 세션이 활성화되었다가, 사용자가 로그아웃하거나 일정 시간 동안 활동이 없으면 서버에서 세션 데이터가 삭제됩니다. 즉, 세션은 브라우저 세션 동안만 유지되며 사용자의 설정이 아닌 사용자의 활동에 따라 수명이 제어됩니다.<br><br>

<br>

###### Q. 쿠키와 세션의 용도 차이를 좀더 구체적으로 설명해줘 ❗<br><br>

쿠키는 주로 클라이언트 측에서 사용자의 상태, 개인화된 설정, 추적 정보 등을 저장하는 데 사용됩니다.<br>

예를 들어, 로그인 정보, 언어 및 지역 설정, 장바구니 내용을 저장하는 데 용이합니다.<br><br>

세션은 저장 위치가 서버이기 때문에 사용자 접근 측면에서 비교적 안전하기 때문에 사용자의 로그인 상태, 장바구니 내용, 웹 애플리케이션 내에서의 상태 정보와 같이 민감한 데이터를 안전하게 저장하기 위해 사용되곤 합니다. 세션은 서버 측에서 관리되므로 데이터를 클라이언트에서 직접 수정할 수 없으며, <mark>일반적으로 더 보안적인 용도에 사용</mark>됩니다. 얼핏 생각하기에는 무조건 세션을 쓰면 좋을 것이라고 생각할 수 있지만, <mark>서버 자원의 낭비를 방지하여 웹 사이트의 속도를 높이는 것이 중요한 상황에서는 쿠키를 사용하는 것이 유리</mark>합니다.<br><br>

<br>

###### Q. 쿠키와 세션을 사용했을 때 얻는 이점 좀더 구체적으로 설명해줘 ❗ <br><br>

미리 스포하면.. 쿠키와 세션은 HTTP 프로토콜의 특징이자 약점을 보완하기 위해서 사용됩니다.<br>

이를 이해하기 위해서는 몇 가지 알아야 하는 사전 지식이 있습니다.<br><br>

HTTP 프로토콜의 특성 01. <mark>Connectionless 프로토콜 (비연결지향)</mark>란, 클라이언트가 서버에 요청(Request)을 했을 때, 그 요청에 맞는 응답(Response)을 보낸 후 연결을 끊는 처리방식을 말합니다.<br><br>

HTTP 프로토콜의 특성 01. <mark>Stateless 프로토콜 (상태정보 유지 안함)</mark>란, 클라이언트의 상태 정보를 가지지 않는 서버 처리 방식을 말합니다. 클라이언트와 첫번째 통신에서 데이터를 주고 받았다 해도, 두번째 통신에서 이전 데이터를 유지하지 않는 것이지요. 하지만 실제로는 데이터 유지가 필요한 경우가 많습니다. 때문에 정보가 유지되지 않으면 매번 페이지를 이동할 때마다 로그인을 다시 하거나, 상품을 선택했는데 구매 페이지에서 선택한 상품의 정보가 없거나 하는 등의 일이 발생할 수 습니다. 이를 보완 즉, Stateless 경우를 대처하기 위해서 쿠키와 세션을 사용하곤 합니다.<br><br>
    
이제 앞서 설명한 쿠키와 세션의 차이점인 상태 정보의 저장 위치을 접목시켜 설명해보겠습니다. 쿠키는 '클라이언트(local)'에 저장하고, 세션은 '서버'에 저장한다고 말씀을 드렸는데, 이는 서버와 클라이언트가 통신을 할 때 통신이 연속적으로 이어지지 않고 한 번 통신이 되면 끊어진다는 의미입니다. 따라서 서버는 클라이언트가 누구인지 매번 번거롭게 인증을 해줘야 하죠. 또한 웹페이지의 로딩을 느리게 만드는 요인이 되기도 합니다. 이때 쿠키와 세션을 사용하면 이러한 번거로움을 보완할 수 있습니다.<br><br>
